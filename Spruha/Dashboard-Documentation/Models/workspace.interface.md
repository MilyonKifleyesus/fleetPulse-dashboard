### WidgetType Union Type
**WHERE**: Lines 1-7

**WHY**: Defines allowed widget types. Ensures only valid widget types can be created.

**WHICH**: Union type pattern

**Values**:
- `'metric-card'`: Displays a single metric
- `'chart-widget'`: Displays charts
- `'table-widget'`: Displays tabular data
- `'activity-feed'`: Shows activity feed
- `'status-indicator'`: Status indicators
- `'custom'`: Custom widget types

---

### WidgetFrame Interface
**WHERE**: Lines 21-32

**WHY**: Core interface for all widgets. Defines widget structure with position, size, and state.

**Properties**:
- `id: string` - Unique widget identifier
- `type: WidgetType` - Widget type (from union above)
- `title: string` - Widget display title
- `icon?: string` - Optional icon identifier
- `position: GridPosition` - Grid position
- `size: GridSize` - Widget dimensions
- `isMinimized: boolean` - Minimized state
- `isMaximized: boolean` - Maximized state
- `config: Record<string, any>` - Widget-specific config
- `order: number` - Display order

**WHEN**: Created when widgets are added, updated when moved/resized

**HOW**: Generated by `WidgetRegistryService` or manually created

---

### GridPosition Interface
**WHERE**: Lines 9-14

**WHY**: Defines widget position in CSS Grid system.

**Properties**:
- `column: number` - Starting column (1-based)
- `row: number` - Starting row (1-based)
- `columnSpan: number` - Columns to span (1-12)
- `rowSpan: number` - Rows to span (1-8)

**HOW**: Converted to CSS `grid-area` value

---

### WorkspaceLayout Interface
**WHERE**: Lines 34-39

**WHY**: Defines grid layout configuration for the workspace.

**Properties**:
- `columns: number` - Number of grid columns
- `gap: number` - Gap between widgets (pixels)
- `breakpoint: string` - Responsive breakpoint name
- `minWidgetWidth: number` - Minimum widget width

---

### WorkspaceState Interface
**WHERE**: Lines 41-49

**WHY**: Complete workspace state persisted to localStorage.

**Properties**:
- `workspaceId: string` - Workspace identifier
- `userId?: string` - Optional user identifier
- `widgets: WidgetFrame[]` - Array of all widgets
- `layout: WorkspaceLayout` - Layout configuration
- `lastModified: Date | string` - Last modification timestamp
- `version: string` - State version for migration
- `isEditMode: boolean` - Current edit mode state

**WHEN**: Saved to localStorage after any widget change

**HOW**: Serialized to JSON for localStorage storage
